## 📋 笔记创建核心原则

### 1. 内容保留原则
- ✅ **知识点和概念**：原封不动保留书本内容
- ✅ **公式和定理**：完整记录，包括条件和结论
- ✅ **例题**：保留有意义的例题及详细解答
- ✅ **定义**：精确记录数学定义，标注原书页码

### 2. 标签化原则
每个笔记必须包含完整的标签体系，以实现与题库的无缝对接：

```markdown
## 🏷️ 标签体系
- **L1知识点**：`具体知识点1` `具体知识点2` ...
- **L2方法**：`解题方法1` `解题方法2` ...
- **L3章节**：`章节名称`
- **难度**：`easy` / `medium` / `hard` / `expert`
```

### 3. 结构化原则
- 每个知识点独立成节
- 包含定义、理解要点、例题、应用
- 建立知识点之间的内部链接

## 🔄 笔记生成具体流程

### 第一步：读取PDF内容
```python
# 使用pdf-reader读取对应页面
pdf-reader:read_pdf(file_path, page_numbers=[起始页, 结束页])
```

### 第二步：分析内容结构
1. 识别知识点层级（一级标题、二级标题等）
2. 标记定义、定理、公式位置
3. 定位例题及解答
4. 记录原书页码

### 第三步：创建笔记结构
1. 使用标准模板
2. 填入标题和标签
3. 按照原书顺序组织内容
4. 保持数学符号的准确性

### 第四步：内容填充原则
- **定义部分**：完整复制，标注"原书第X页"
- **理解要点**：提炼关键点，用简洁语言
- **公式定理**：使用LaTeX格式，确保准确
- **例题部分**：包含题目、解答过程、标注"原书例题"
- **易错提醒**：基于常见错误总结

### 第五步：质量检查
- 检查LaTeX公式渲染
- 验证标签准确性
- 确认内部链接有效
- 核对原书页码

## 📝 统一格式规范

### 1. 标题规范
- 一级标题：`# 知识点名称`
- 二级标题：`## 一、主要内容`（使用中文数字）
- 三级标题：`### 1. 具体内容`（使用阿拉伯数字）
- 特殊标题：`### 📖 定义` `### 💡 理解要点` `### 🔑 重要公式`

### 2. 图标使用规范
- 📖 定义/概念
- 💡 理解要点/技巧
- 🔑 重要公式/关键
- 📐 例题/练习
- 🎯 易错提醒/注意
- 📊 图表/数据
- 🔗 相关链接
- ⚠️ 警告/重要提醒
- ✅ 完成/正确
- ❌ 错误/不要做

### 3. 公式格式规范
- 行内公式：`$公式$`
- 独立公式：`$$公式$$`
- 公式编号：手动标注如 (1.1)
- 多行公式：使用`aligned`环境

### 4. 例题格式规范
```markdown
### 📐 例题X.X（原书例题，YYYY年真题）
[题目内容]

**解**：[可选：解法一/解法二]
[详细解答过程]

**评注**：[可选的解题技巧或注意事项]
```

## ⚠️ 重要注意事项

### 1. 内容准确性
- **数学符号**：确保所有数学符号准确无误
- **公式格式**：LaTeX格式必须能正确渲染
- **原书还原**：定义、定理必须与原书一致
- **页码标注**：每个重要内容都要标注原书页码

### 2. 标签一致性
- 必须使用与题库完全一致的标签
- 不要创造新标签，使用已定义的标签
- 多个标签用反引号分隔

### 3. 结构完整性
- 每个笔记都要有完整的标签体系
- 重要的定理、公式不能遗漏
- 典型例题必须包含

### 4. 链接规范
- 使用`[[双方括号]]`创建内部链接
- 链接名称要与实际文件名一致
- 及时建立知识点之间的关联

## 🔧 常见问题处理

### 1. 公式太长
- 使用`aligned`环境分行
- 适当添加空格提高可读性

### 2. 例题过多
- 选择最典型的2-3个例题
- 其他例题可以简要提及

### 3. 内容交叉
- 使用内部链接关联
- 避免重复详细说明

### 4. 原书错误
- 保留原书内容
- 在评注中说明正确内容
